# 目录到代码映射

## 文档目的

本文提供当前仓库的目录地图，帮助维护者快速定位代码边界和关键入口。

## 当前主目录地图

### 核心执行系统

- `src/agent/`
  - turn、session、checkpoint、verification、acceptance、runtime stats、provider 请求
- `src/objective/`
  - 当前目标 key 与任务元数据标记；不做自动分析、规划、路由或派发
- `src/capabilities/tools/`
  - 内建工具、registry、governance、runtime exposure
- `src/documents/`
  - 文档型能力与工具支撑
- `src/capabilities/skills/`
  - skill 发现、匹配、加载与能力提示
- `src/capabilities/dreaming/`
  - Dreaming 单轮 Mirror World 执行、Dreaming Loop 状态账本、时间戳 id
- `src/capabilities/workflows/`
  - workflow capability package，包括 `workflow.dreaming-loop`
- `src/capabilities/mcp/`
  - 外部能力接入与 registry integration
- `src/control/ledger/`
  - SQLite 控制面账本
- `src/tasks/`
  - task 领域逻辑
- `src/capabilities/team/`
  - 队友、消息总线、协同协议
- `src/execution/`
  - execution lifecycle、worker、closeout、reconcile
- `src/worktrees/`
  - worktree 创建、索引和移除
- `src/config/`
  - config file、runtime config、env 装载
- `src/observability/`
  - 事件、crash log、doctor 报告

### 宿主运行边界

- `src/host/`
  - 共享 session 入口
  - 共享 turn 入口
  - host extra tool 注入

### 宿主适配层

- `src/cli/`
  - 命令入口、one-shot、session 命令
- `src/interaction/`
  - 共享交互驱动
- `src/ui/`
  - interactive shell、local commands、spinner、runtime summary、todo styling 等 CLI 壳层辅助
- `src/runtime-ui/`
  - 统一终端渲染、前台 execution stream、tool/dispatch/result 语义输出、身份中文映射与颜色策略
- `src/shell/cli/`
  - CLI 展示层
- `src/telegram/`
  - Telegram 服务、turn runner、附件与投递

## 当前使用方式

阅读代码时，建议先定位：

1. 真相源是否落在 `src/agent/session/`、`src/control/ledger/` 或 `src/config/`
2. 主路径是否经过 `src/agent/turn/` 与 `src/host/`
3. 扩展是否经过 `src/capabilities/tools/`、`src/capabilities/skills/`、`src/capabilities/mcp/` 或 `src/host/toolRegistry.ts`

这样可以更快判断一个改动是否符合当前架构边界。

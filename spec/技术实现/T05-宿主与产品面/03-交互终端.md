# 交互终端

## 文档目的

本文说明 interactive host 当前负责什么，以及它与 `src/host/`、展示层和本地命令之间的边界。

## 模块目标与当前状态

交互终端当前是 CLI 下的宿主外壳，负责本地输入输出、turn 展示和中断语义，不承担核心状态裁决。

## 正式边界

主要代码边界：

- `src/ui/interactive.ts`
- `src/interaction/`
- `src/shell/cli/`

## 主路径

当前主路径如下：

1. interactive host 接收用户输入或本地命令。
2. session 与 turn 通过 `src/host/` 进入核心。
3. 结果先进入 `src/runtime-ui/` 的统一运行 UI 事件层，再由终端外壳输出。
4. interrupt、quit 和 summary 语义在宿主层处理，但不改核心真相源。

## 模块职责与非职责

### 交互终端负责

- 本地输入输出
- turn 展示
- interrupt 与 exit guard
- 运行摘要展示

### 交互终端不负责

- 直接调用 raw turn
- 保存平行 runtime 状态
- 改写Lead/机器边界或真相源规则

## 测试与验证

当前主要由以下测试保护：

- `tests/interaction/interaction-shell.part*.test.ts`
- `tests/host/host-runtime-boundary.test.ts`

## 当前落地决定

interactive 当前继续作为宿主外壳存在；其职责边界固定在“交互与展示”，而不是“定义核心运行规则”。
交互终端只负责承载 runtime-ui 输出，不再维护自己的工具渲染门户。
terminal log 记录降噪后的稳定终端现场；动态 spinner 帧、光标刷新帧等 transient terminal frame 不写入观测日志。
